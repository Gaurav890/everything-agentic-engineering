# Frontend rules

- Start from product context and a visual thesis, not component generation.
- Read `docs/20-design/DESIGN_SYSTEM.md`.
- Design loading, empty, error, sparse, dense, disabled, and success states.
- Use semantic HTML and visible focus states.
- Respect reduced-motion preferences.
- Make URL state deep-linkable when appropriate.
- Verify important pages at mobile, tablet, and desktop breakpoints.
- Use Playwright to inspect actual rendered output.
- Builder and evaluator must be separate roles for substantial UI work.
